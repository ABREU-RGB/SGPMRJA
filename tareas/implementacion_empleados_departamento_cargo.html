1 Contexto del Problema

En el módulo de Empleados de SGPMRJA, los campos cargo y departamento se almacenaban como texto libre (varchar(100)) directamente en la tabla empleado, sin catálogos maestros propios ni relación entre ellos.

PROBLEMAS IDENTIFICADOS

  • Inconsistencias de datos: "Producción" vs "Produccion" vs "PRODUCCION" — cada variante se guardaba como valor único.
  • Sin vínculo lógico: un empleado podía tener "Costurera" en "Administración", lo cual viola reglas de negocio.
  • Renombrado imposible: cambiar el nombre de un departamento requería actualizar cada empleado uno por uno.
  • Sin catálogo de valores: imposible enumerar cargos y departamentos disponibles para auditoría o reportería.
  • Falta de historial: no había forma de inhabilitar valores obsoletos preservando datos históricos.

Un departamento tiene muchos cargos. Un cargo pertenece a un único departamento. Esto enforza la regla de negocio: un cargo siempre está contextualizado por su área operativa.

Departamento (select primario)

  • Al crear un cargo nuevo, queda automáticamente vinculado al departamento activo.

6 Fix Crítico: Modales Anidados en Bootstrap 5

BUG DETECTADO

Al abrir un modal pequeño (ej: "Nuevo Departamento") desde dentro de otro modal (ej: "Agregar Empleado") y cerrar el hijo con X o Cancelar, también se cerraba el modal padre, haciendo perder todos los datos cargados en el formulario de empleado.

Causa: Bootstrap 5 no soporta modales apilados nativamente (según su propia documentación oficial). Al cerrar cualquier modal, Bootstrap remueve modal-open del body de forma incondicional, afectando al padre.

SOLUCIÓN APLICADA

Se adoptó el patrón modal-hidden-temp ya existente en el módulo Inventario pero se aplicó de forma global en resources/views/admin/layouts/app.blade.php, cubriendo automáticamente todos los módulos del proyecto.

Código del patch global

// Al abrir un modal sobre otro, ocultar el padre temporalmente $(document).on('show.bs.modal', '.modal', function () { var $abiertos = $('.modal.show').not(this); if ($abiertos.length > 0) { $abiertos.addClass('modal-hidden-temp'); $(this).data('parentModals', $abiertos); } }); // Al cerrar el hijo, restaurar el padre y el scroll-lock $(document).on('hidden.bs.modal', '.modal', function () { var $padres = $(this).data('parentModals'); if ($padres && $padres.length > 0) { $padres.removeClass('modal-hidden-temp'); } if ($('.modal.show').length > 0) { $('body').addClass('modal-open'); } });

Por qué funciona

  • La clase modal-hidden-temp aplica opacity: 0 y pointer-events: none al padre.
  • La instancia del modal padre sigue viva en Bootstrap — nunca se llama a su método hide().
  • El usuario solo ve el modal hijo mientras interactúa con él.
  • Al cerrar el hijo, se quita la clase y el padre reaparece intacto con todos sus datos.

COBERTURA AUTOMÁTICA

Al estar ubicado en el layout global, este fix protege automáticamente todos los módulos del proyecto presentes y futuros que usen modales anidados: Empleados (showModal → addDepartamento/addCargo), Productos (showModal → addTipo), Inventario (createModal → modalAddInsumo), etc.
